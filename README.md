## Валидации (5 обязательных)

1. **Ссылочная целостность** — pvz_id должен существовать в таблице pvz
2. **Корректность расписания** — open_time < close_time; weekday ∈ {0..6}
3. **Типы операций** — type ∈ {in, out, return}
4. **Рабочие часы** — operations.ts должен попадать в открытые часы ПВЗ
5. **День без расписания** — операции в дни без расписания запрещены

Все ошибки валидации логируются в таблицу `error_log`.

## API Endpoints

| Метод | URL | Описание |
|-------|-----|----------|
| POST | `/api/login` | Авторизация пользователя |
| POST | `/api/logout` | Выход из системы |
| GET | `/api/me` | Получение информации о текущем пользователе |
| GET | `/api/pvz` | Получение списка ПВЗ (с учётом роли) |
| GET | `/api/pvz/<id>` | Получение деталей ПВЗ и расписания |
| GET | `/api/regions` | Получение списка регионов |
| GET | `/api/operations` | Получение списка операций с фильтрами |
| POST | `/api/operations` | Добавление новой операции |
| GET | `/api/report/load` | Генерация отчёта о загрузке ПВЗ |
| GET | `/api/report/heatmap` | Генерация тепловой карты загрузки |
| GET | `/api/export/csv` | Экспорт данных в CSV |
| GET | `/api/errors` | Получение журнала ошибок данных |

> Подробная спецификация API доступна в файле [api/openapi.yaml](api/openapi.yaml).

### Фильтры для `/api/report/load`:
- `pvz_id` — конкретный ПВЗ
- `region` — регион
- `type` — тип операции (in/out/return)
- `date_from` / `date_to` — диапазон дат (YYYY-MM-DD)

## KPI

- **Средняя загрузка** = среднее(ops_per_hour / capacity_per_hour) за период
- **Загрузка в часы пик** = средняя загрузка в 12–14 и 18–20 ч.
- **Доля перегрузок** = кол-во интервалов с load > 100% / всего интервалов

---

## Демо-сценарий

1. Войти как `analyst1` / `anapass`
2. На дашборде посмотреть KPI и графики за Март 2025
3. Перейти в «Отчёт загрузки», отфильтровать по ПВЗ №1
4. В «Тепловой карте» выбрать ПВЗ и построить карту по часам/дням
5. Экспортировать CSV через кнопку «CSV»
6. Добавить операцию через меню «Добавить операцию» — проверить валидацию (попробовать указать нерабочий час)
7. Просмотреть «Журнал ошибок»
8. Выйти и войти как `operator1` — убедиться, что доступен только ПВЗ №1
