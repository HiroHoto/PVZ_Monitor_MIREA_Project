# BPMN-диаграмма (текстовое описание)

## Участники процесса
- Оператор ПВЗ
- Супервайзер региона
- Операционный аналитик

## Последовательность шагов
1. Добавление операции → 2. Проверка пропускной способности → 3. Уведомление о перегрузке

```mermaid
graph TD
    Start((Начало)) --> AddOp[Добавление операции]
    AddOp --> CheckCap{Проверка пропускной способности}
    CheckCap -- Ок --> Log[Регистрация операции]
    CheckCap -- Превышение --> Notify[Уведомление о перегрузке]
    Notify --> Log
    Log --> End((Конец))
    
    subgraph "Альтернативные сценарии"
        Critical{Перегрузка > 120%?}
        Critical -- Да --> Block[Блокировка/Перенаправление]
        Block --> End
    end
    CheckCap -.-> Critical
```

## Альтернативные потоки
- Если пропускная способность превышена, то отправить уведомление.

## Альтернативные сценарии
- Отмена операции при критической перегрузке
- Перенаправление операций на соседний ПВЗ
- Автоматическая блокировка новых операций при перегрузке >120%
